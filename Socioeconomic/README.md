# North Carolina Exposome Knowledge Graph

## Overview

This project constructs a comprehensive knowledge graph integrating environmental, socioeconomic, and health data for all 100 counties in North Carolina. The knowledge graph uses real data from US Census Bureau and County Health Rankings, enhanced with semantic web ontology mappings for interoperability with established health informatics systems.

**Key Features:**
- 100 North Carolina counties with 16+ indicators each
- Real data from Census 2022 and County Health Rankings 2025
- Semantic web ontology mappings (ExO, ECTO, ENVO, NCIT)
- Multiple RDF export formats (Turtle, N-Triples, RDF/XML)
- 28 example SPARQL queries for data exploration
- Statistical correlation analysis between indicators

### Ontology Enhancement

#### `enhance_kg_with_ontologies.py`
**Type:** Python Script
**Purpose:** Add semantic web ontology mappings to the knowledge graph
**What it does:**
- Loads base knowledge graph data from CSV
- Maps indicators to established ontologies:
  - **ExO** (Exposure Ontology) - Exposure classifications
  - **ECTO** (Environmental Conditions) - Environmental/social exposures
  - **ENVO** (Environment Ontology) - Environmental contexts
  - **NCIT** (NCI Thesaurus) - Health outcomes
- Adds dataset provenance with PROV-O
- Uses RDF Data Cube vocabulary for observations
- Exports to three RDF formats

**Ontology Mappings:**
| Indicator | Ontology Class | Description |
|-----------|---------------|-------------|
| obesity_pct | ncit:C3283 | Obesity (NCI Thesaurus) |
| diabetes_pct | ncit:C2985 | Diabetes Mellitus |
| pm25 | ecto:0000460 + envo:00002005 | PM2.5 in air |
| poverty_rate | ecto:0000095 | Poverty exposure |
| median_income | ecto:0000090 | Income exposure |

**How to Use:**
```bash
# Install RDF library
pip install rdflib

# Run enhancement
python3 enhance_kg_with_ontologies.py
```

**Input Required:**
- `nc_exposome_data.csv` (generated by notebook)

**Output:**
- `nc_exposome_kg_ontology.ttl` - Turtle format (most readable)
- `nc_exposome_kg_ontology.nt` - N-Triples format
- `nc_exposome_kg_ontology.rdf` - RDF/XML format

**Statistics:**
- 15,469 total RDF triples
- 100 counties
- 16 indicators with ontology mappings
- 1,599 observations
- 75 statistical correlations

#### `SPARQL_EXAMPLES.md`
**Type:** Markdown
**Purpose:** 28 example SPARQL queries for knowledge graph exploration
**Contents:**
- Basic queries (list counties, indicators)
- Environmental-health associations
- Socioeconomic-health pathways
- County comparisons
- Statistical analysis
- Data provenance queries
- Advanced analytical queries

**Query Categories:**
1. **Basic Queries (3)** - Fundamental data retrieval
2. **Environmental-Health (2)** - PM2.5 correlations
3. **Socioeconomic-Health (3)** - Poverty, education, unemployment
4. **County Comparison (2)** - Urban vs rural, rankings
5. **Statistical Analysis (3)** - Correlations, risk factors
6. **Data Provenance (2)** - Source attribution
7. **Advanced Analytics (3)** - Multi-dimensional risk

**Example Query:**
```sparql
# Find counties with high PM2.5 and diabetes
PREFIX ex: <http://example.org/nc-exposome/>
PREFIX schema: <http://schema.org/>

SELECT ?countyName ?pm25 ?diabetes
WHERE {
  ?county schema:name ?countyName .
  ?obs1 schema:location ?county ;
        ex:measuredIndicator ex:indicator/pm25 ;
        schema:value ?pm25 .
  ?obs2 schema:location ?county ;
        ex:measuredIndicator ex:indicator/diabetes_pct ;
        schema:prevalence ?diabetes .
  FILTER(?pm25 > 8.0 && ?diabetes > 11.0)
}
```

## Use Cases

### 1. Explore Health Disparities
**Tools:** Jupyter notebook, visualizations
**How:** Run the notebook and examine correlation heatmaps, cluster analysis, and county comparisons to identify patterns in health disparities.

### 2. Query Specific Relationships
**Tools:** Python + rdflib, SPARQL queries
**How:** Use the example queries in `SPARQL_EXAMPLES.md` to find relationships between socioeconomic factors and health outcomes.

**Example:**
```python
from rdflib import Graph

g = Graph()
g.parse("nc_exposome_kg_ontology.ttl", format="turtle")

query = """
PREFIX ex: <http://example.org/nc-exposome/>
SELECT ?county ?poverty ?diabetes
WHERE {
  ?obs1 schema:location ?county ;
        ex:measuredIndicator ex:indicator/poverty_rate ;
        schema:value ?poverty .
  ?obs2 schema:location ?county ;
        ex:measuredIndicator ex:indicator/diabetes_pct ;
        schema:prevalence ?diabetes .
  FILTER(?poverty > 15)
}
"""

for row in g.query(query):
    print(f"County: {row.county}, Poverty: {row.poverty}%, Diabetes: {row.diabetes}%")
```

## Technical Architecture

### Knowledge Graph Structure

**Node Types:**
- **Location** - Counties (100 nodes)
- **Indicator** - Socioeconomic and health measures (16 nodes)
- **Measurement** - Actual values (~1,600 nodes)
- **Correlation** - Statistical relationships (75 nodes)

**Edge Types:**
- **rdf:type** - Type classification
- **schema:location** - Links measurement to county
- **ex:measuredIndicator** - Links measurement to indicator
- **ex:indicator1/indicator2** - Links correlation to indicators
- **prov:wasDerivedFrom** - Provenance tracking

**Provenance Tracking:**
- Each triple includes source attribution
- Confidence scores based on data source
- Temporal information (year)
- Statistical metadata (p-values, sample size)

## Troubleshooting

### Issue: Census API fails
**Solution:**
- Check API key in `nc_exposome_kg.ipynb`
- Notebook will automatically fall back to 2021 data
- Verify internet connection

### Issue: CHR data file not found
**Solution:**
- Ensure `chr_2025_nc_complete.csv` is in project directory
- Or run `parse_chr_complete.py` if you have the Excel file
- Notebook will fall back to CDC PLACES data if CHR unavailable

### Issue: RDF parsing errors
**Solution:**
- Update rdflib: `pip install --upgrade rdflib`
- Check file encoding (should be UTF-8)
- Verify file is complete (not truncated)

### Issue: Missing visualizations in notebook
**Solution:**
- Ensure matplotlib backend is configured: `%matplotlib inline`
- Install plotly: `pip install plotly`
- Restart kernel: Kernel → Restart & Run All
